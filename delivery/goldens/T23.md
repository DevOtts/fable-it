# Golden T23 — cost-aware delegation routing (E1, tabletop, G2.5/SD10)

**Registered:** 2026-07-02, before E1 implementation.

## Scenario setup
Step 3 decomposition of a fable-it run yields three work packets:
1. a **grep-sweep packet** — find every call site of a deprecated helper across
   the repo (mechanical, low-ambiguity, easily verified);
2. an **implementation packet** — build a feature against a locked spec;
3. the **fresh-context verifier** — audit the draft report against `evidence.md`.
The session model is Opus 4.8. The host supports per-agent model selection
(cheap = Haiku 4.5, mid = Sonnet 5, top = Opus 4.8).

## Expected transcript
1. The model applies the delegation routing rule by reference to the canonical
   tier table (spec §4.1) — it does not invent ad-hoc tiers or copy the table.
2. Routing: grep-sweep → cheap tier (mechanical/high-volume); implementation →
   mid tier (standard build against a locked contract); verifier → top tier —
   the never-downgrade list explicitly protects the verifier (and anything
   writing to `decisions.md` / locking interfaces).
3. No packet silently defaults to the top tier: each of the three gets an
   explicit tier choice with a one-line reason logged to
   `.taskstate/run-memory.md`. When genuinely unsure, the rule is
   default-inherit the session model — stated, not silent.
4. The final report's cost line is a per-agent table: packet · model tier · why.

## Pass rule (binding, from epics doc)
Sweep → cheap tier, implementation → mid, verifier → top (never downgraded);
each choice + reason in `run-memory.md`; report shows the per-agent cost table;
PASS iff no packet defaults silently to top tier and the verifier is not
downgraded.
