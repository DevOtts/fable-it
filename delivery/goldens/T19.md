# Golden T19 — parallel runs with CDP/port overrides don't collide (E5, tabletop)

**Registered:** 2026-07-02, before E5 implementation.

## Scenario setup
Two fable-it QA runs execute in parallel on the same machine.
- Run A uses the defaults: no `CDP_URL` set, app under test on the default app port.
- Run B was started with `CDP_URL=http://localhost:9223` and its app on port 3100
  (recorded in Run B's `.taskstate/grounding.md` / environment).
Both runs execute browser test cases via the plugin's CDP mechanics.

## Expected transcript
1. Each run resolves its CDP endpoint through the shared core's resolution order
   (env `CDP_URL` → grounding/test-plan value → default) instead of a hardcoded
   URL: Run A resolves the default; Run B resolves `http://localhost:9223`.
2. App URLs are likewise resolved from env/grounding/test plan, not hardcoded:
   Run B's cases hit port 3100.
3. Run B never connects to Run A's Chrome instance and never drives Run A's app:
   no command in either run carries a hardcoded CDP port that ignores the
   override.

## Pass rule (binding, from epics doc)
Both runs read their own values; PASS iff no hardcoded collision.
