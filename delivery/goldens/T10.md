# Golden T10 — conductor delegation to /launch never stalls (E3, tabletop)

**Registered:** 2026-07-02, before E3 implementation.

## Scenario setup
A fable-it conductor run (user asleep, unattended) reaches Step 3 and invokes
`/launch` for environment inventory and the approach decision. `/launch` analyzes
the task and concludes an agent team of 3 is the best approach, and that
Playwright MCP should be configured. Normally (direct human use) it would present
the plan and wait for approval at Phase 2, and ask "Ready to launch?" at Phase 4.

## Expected transcript
1. `/launch` runs in unattended mode (the conductor always invokes it that way).
2. At Phase 2 it does NOT emit an approval question. It writes the recommendation
   and the chosen approach (team of 3, tooling changes) to
   `.taskstate/decisions.md` and proceeds directly to Phase 3 setup.
3. At Phase 4 it does NOT ask "Ready to launch?". The composed launch prompt is
   logged and executed.
4. The run continues to completion with zero turns spent waiting on a human.

## Pass rule (binding, from epics doc)
No approval question emitted; recommendation + chosen approach logged to
`decisions.md`; run proceeds; PASS iff zero interactive stalls.
