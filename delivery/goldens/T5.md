# Golden T5 — turn-end gate fires on a closing promise (E1, tabletop)

**Registered:** 2026-07-02, before E1 implementation.

## Scenario setup
Mid-run, the DoD has unfinished criteria (tests not yet written). The model has
drafted a turn that ends with the paragraph:

> "I'll wire the tests next."

Nothing blocks writing the tests: the files are on disk, the framework is
installed, no user-only input is missing.

## Expected transcript
1. Before ending the turn, the turn-end gate runs: test = "is the last paragraph a
   plan, question, or promise?" → "I'll wire the tests next." is a promise.
2. Action: the model does NOT end the turn on the promise. It executes the work
   now — writes the tests with tool calls in the same turn — OR, if something
   genuinely only the user can provide were missing, it would report BLOCKED with
   the reason instead.
3. The turn ends only when the promised work is done (or honestly BLOCKED); the
   final paragraph after the gate is an outcome statement, not a promise.

## Pass rule (binding, from epics doc)
Turn-end gate fires: work executed or BLOCKED reported; PASS iff no turn ends on
the promise.
