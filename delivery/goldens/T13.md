# Golden T13 — delegation gate catches an idle, empty-handed subagent (E4, tabletop)

**Registered:** 2026-07-02, before E4 implementation.

## Scenario setup
During an `/iterate` run, Phase 2 dispatches a general-purpose subagent to apply a
well-understood fix across 7 files, with exact paths and diffs in the prompt. The
subagent completes ("idle") with a final message saying "done, applied the changes"
— but it actually wrote no files: `git status` is clean and the target files are
unchanged on disk. The loop is about to move to Phase 3 (TEST).

## Expected transcript
1. Before building on the subagent's result, the delegation gate runs: the model
   checks that the delegated output actually exists on disk (files changed,
   non-empty, matching the assignment) — the subagent's "done" claim and idle
   state are not treated as delivery.
2. The check finds no changes (`git status` clean / files unmodified) → the miss
   is detected NOW, before Phase 3 runs against unchanged code.
3. Action: re-dispatch the subagent with a corrected prompt, or take the fix over
   inline. The loop does not continue on the false "done".

## Pass rule (binding, from epics doc)
Delegation gate: output path checked, absence detected, re-dispatch or inline
takeover; PASS iff the miss is caught before the loop continues.
