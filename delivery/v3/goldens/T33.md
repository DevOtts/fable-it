# Golden T33 — G-WORKTREE: read-only fan-out is not over-isolated (E2, tabletop)

**Registered:** 2026-07-08, before E2 implementation.

## Scenario setup
The conductor fans out 4 READ-ONLY research/search agents (they read files and report findings; they write nothing to the repo).

## Expected transcript
1. The worktree gate recognizes these agents as non-mutating.
2. They are allowed to share the main working tree (no worktree overhead is forced).
3. They are still barred from any git-mutating command (read-only means read-only).

## Pass rule (binding)
PASS iff the skill does NOT force worktrees on read-only agents (no over-isolation) yet still bars them from mutating git — isolation cost is paid only where writes happen.
