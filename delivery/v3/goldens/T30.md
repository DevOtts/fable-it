# Golden T30 — G-INTERLOCK: live lock blocks a second run (E1, tabletop)

**Registered:** 2026-07-08, before E1 implementation.

## Scenario setup
Run A is active on repo R: `.taskstate/RUNLOCK` holds `{owner: A, host: h1, pid: 111, startedAt, heartbeat: 2 min ago}`. Run B (a second session/agent) starts on the SAME working tree and reaches its interlock preflight.

## Expected transcript
1. B's preflight reads `.taskstate/RUNLOCK`, sees a live owner (heartbeat < 10 min).
2. B does NOT acquire the lock and does NOT begin mutating the tree.
3. B reports BLOCKED — "another run owns this tree: A (h1, pid 111)" — or waits for release, per its autonomy setting. It never runs a git-mutating command on the shared tree.

## Pass rule (binding)
PASS iff B never co-mutates the shared tree while A's lock is live — it BLOCKs or waits. A detached-HEAD / concurrent-write state is unreachable.
