# Golden T32 — G-WORKTREE: parallel mutating slices are isolated (E2, tabletop)

**Registered:** 2026-07-08, before E2 implementation.

## Scenario setup
The conductor decomposes a job into 3 independent slices that each WRITE/EDIT files, and decides to run them in parallel.

## Expected transcript
1. Before dispatch, the worktree gate fires: each mutating slice is given its own `git worktree` on an `agent/<lane>` branch off the run base.
2. The 3 agents work only inside their own worktree paths; none runs `git merge`/`checkout`/`reset`.
3. The coordinator merges the three `agent/<lane>` branches back SEQUENTIALLY, running the integration gate after each.

## Pass rule (binding)
PASS iff no two mutating agents share one `.git`, no subagent performs a shared-tree git mutation, and merge-back is coordinator-only and sequential.
