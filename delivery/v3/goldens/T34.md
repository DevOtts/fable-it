# Golden T34 — G-INTEGRATE: integration catches an isolation-green slice (E3, tabletop)

**Registered:** 2026-07-08, before E3 implementation.

## Scenario setup
A parallel slice adds a new `package.json` and passes its own tests inside its isolated worktree. It has NO lockfile. On merge-back to the base, CI-shape (install from lockfile) would break (EC-G5).

## Expected transcript
1. The slice merges back; the delegation gate confirms its output exists.
2. The integration gate then runs the project's build/lockfile/test shape on the MERGED tree.
3. The missing lockfile is detected → the slice is REOPENED (not accepted); the wave is not marked complete on "output exists" alone.

## Pass rule (binding)
PASS iff a slice green-in-isolation but integration-broken is reopened, and no wave is accepted on existence alone — acceptance requires integration.
