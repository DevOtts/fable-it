# Golden T31 — G-INTERLOCK: stale lock is reclaimed (E1, tabletop)

**Registered:** 2026-07-08, before E1 implementation.

## Scenario setup
`.taskstate/RUNLOCK` exists but its owner is dead: heartbeat 40 min old, pid no longer running. A new run starts.

## Expected transcript
1. Preflight reads the lock, computes heartbeat age > 10 min (and/or pid absent) → classifies it STALE.
2. The run reclaims the lock, writing a `run-memory.md` note: "reclaimed stale RUNLOCK from <owner>, heartbeat <age>".
3. The run proceeds normally, now holding a fresh lock.

## Pass rule (binding)
PASS iff a stale lock never permanently blocks a run AND the reclaim is logged (never a silent takeover).
